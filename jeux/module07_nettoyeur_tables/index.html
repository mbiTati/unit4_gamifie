8 questions — Testez vos connaissances !

1.

genres contient plusieurs valeurs (rock,jazz) dans une cellule → violation 1NF. telephones contient plusieurs numéros → violation 1NF. Ces colonnes ne sont pas atomiques.

2.

Extraire genres dans une table associative GENRE_ARTISTE. Garder un seul téléphone par artiste (ou créer table TELEPHONE_ARTISTE si plusieurs nécessaires).

3.

Atomicité 1NF = chaque cellule contient UNE SEULE valeur indivisible. Atomicité ACID = une transaction est TOUT OU RIEN (réussit complètement ou échoue complètement). Deux concepts TOTALEMENT DIFFÉRENTS malgré le même mot.

4.

Violation 3NF : si SESSION avait (id_session, id_salle, nom_salle, tarif_salle). id_session → id_salle → nom_salle est transitive. Correction : garder seulement →id_salle dans SESSION, nom_salle et tarif restent dans SALLE.

5.

Stocker nom_artiste dans FACTURE pour éviter un JOIN lors de l'impression. Avantage : requête plus rapide. Inconvénient : si l'artiste change de nom, il faut mettre à jour toutes ses factures. Choix conscient et documenté.
