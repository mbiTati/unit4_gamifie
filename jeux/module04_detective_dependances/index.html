Dépendances fonctionnelles & graphe
8 questions — Testez vos connaissances !

1.

id_artiste → nom. id_artiste → email. id_artiste → telephone. id_album → titre. id_album → prix. id_album → id_artiste. id_salle → nom_salle. id_session → date_session.

2.

Matrice 6×6 : id_artiste détermine nom et email (1). id_album détermine titre, prix et id_artiste (1). Les sources sont id_artiste et id_album (colonnes sans 1).

3.

Sources = id_artiste (détermine nom, email, tel) et id_album (détermine titre, prix). Ce sont les futures PK des entités ARTISTE et ALBUM.

4.

1) id_session → id_salle → nom_salle : transitive car nom_salle dépend de id_salle, pas de id_session. Anomalie : si on renomme une salle, il faut modifier toutes les sessions. 2) id_album → id_artiste → nom_artiste : transitive. Si on stocke nom_artiste dans album, modifier le nom = modifier tous les albums.

5.

Graphe : id_artiste (source) → nom, prenom, email, telephone. id_album (source) → titre, date_sortie, prix. id_album → id_artiste (relation). id_salle (source) → nom_salle, capacite, tarif.
